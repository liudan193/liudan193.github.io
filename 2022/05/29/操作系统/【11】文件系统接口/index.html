【11】文件系统接口

文件的概念

操作系统对存储设备的物理属性加以抽象，从而定义逻辑存储单位，即文件。

文件由操作系统映射到物理设备上；

文件是记录在外存上的相关信息的命名组合；

文件信息有创建者定义；

文件属性

文件被命名以方便人类用户，并且可以通过名称引用；当文件被命名之后，他就独立于进程、用户和系统；

文件属性一般包括：

  • 名称
  • 标识符：唯一标记，用来标记文件系统的文件；
  • 类型
  • 位置
  • 尺寸
  • 保护
  • 时间、日期和用户标识：文件创建、最后修改和最后使用的相关信息等；

所有文件的信息保存在目录结构中，该目录结构也保存在外存上；

文件操作

6种文件操作

文件为抽象的数据类型，要考虑可以对文件执行的操作；系统可以通过系统调用，来创建、写入、读取、重新定位、删除和截断文件；

操作系统如何执行这6个操作：

  • 创建文件：两个步骤，1.在文件系统中为文件找到空间，2.再在目录中创建新文件的条目；
  • 写文件：系统调用提供文件名称和要写入的信息；1.对于给定的文件名称，系统搜索目录以查找文件位置，2.系统应保留写指针（用于指向需要进行下次写操作的文件位置），当写入时，写指针必须被更新；
  • 读文件：系统调用提供文件名称和需要文件的下一个块应该放在哪里；1.系统搜索目录找到相关条目；2.系统保留一个读指针，指向要进行下一次读取操作的文件位置，一旦发生了读取，读指针必须被更新；

因为进程通常需要进行写文件和读文件，所以当前操作位置可以作为进程当前文件位置指针，使用相同的指针，可以解释空间并降低系统的复杂度

  • 重新定位文件：搜索目录找到适当的条目，并且将当前文件位置指针重新定位到给定值；这个操作也叫做文件定位；
  • 删除文件：1.在目录中搜索给定名称的文件，2.找到之后，释放文件空间，3.并删目录条目；
  • 截断文件：删除文件的内容，但是保留它的属性；

搜索目录

由于大多数文件操作都涉及搜索目录，以得到命名文件的相关条目。为了避免不断的搜索，节约时间和资源，许多系统有以下机制：

在首次打开文件操作之前进行系统调用open()，操作系统有一个打开文件表，用于维护所有打开文件的信息；

当请求文件操作时，可以通过该表的索引来指定文件，而不需要搜索；
当文件不在使用时，进程关闭它，同时操作系统删除打开文件表中的该条目；

打开文件

有的系统在首次引用文件时，会隐式地打开它，当结束使用，会自动关闭；

但是大多数操作系统是，使用文件之前，要通过系统调用open()显式地打开它，open()操作：根据文件名，搜索目录，将目录条目复制到打开文件表；

多个进程可以同时打开文件的情况

多个进程可以同时打开文件的情况下：操作系统采用两级的内部表：单个进程表和整个系统表；每个进程表跟踪它打开的所有文件。

单个进程表：单个进程表的每个条目指向整个系统的打开文件表；
系统表：系统表包含与进程无关的信息，如文件在磁盘上的位置、访问日期和文件大小；

系统打开一个文件之后，会在系统表中增加一个这个文件的条目；之后，另一个进程再次打开这个文件，只需在这个进程的进程表中增加一个条目，指向系统表中这个文件的条目即可。

打开计数：系统为打开的每个文件关联一个打开计数，用于表示多少个进程打开了这个文件；

open()会使打开计数加1，close()会使打开计数减1，如果等于0的话，表示不再使用该文件，从系统表中删除这个条目。

文件锁

文件锁：锁定打开的文件，允许一个进程锁定文件，防止其他进程访问它；

共享锁：对个进程可以并发获取；
独占锁：一次只有一个进程可以获取这样的锁；

两种文件锁定的机制：
强制：阻止其他进程访问这个文件；
建议：不会阻止；

文件类型

操作系统应该识别和支持文件类型；

大多数实现文件类型的技术是：将类型作为文件名的一部分，即名称+扩展，如server.c，用.连接；

访问方法

顺序访问

文件信息按顺序加以处理，这种访问比较常见；

文件操作：
read_next();
write_next();

直接访问

文件由固定长度的逻辑记录组成，允许程序按照任意顺序顺序进程读取和写入记录；

文件操作：
read(n);
write(n);

用户提供的一般为相对块号，即相对于文件头的块号；

其他访问方法

创建文件的索引

查找时，1.搜索索引，2.根据指针直接访问文件并且找到所需的记录

目录与磁盘的结构

目录结构是包含所有文件信息的节点集合，目录结构和文件都驻留在磁盘上

一个存储设备可以按照整体来用于文件系统，他也可以细分，以提供更细粒度的控制，例如可以分区；

磁盘或分区可以原始使用，即没有文件系统，或使用文件系统格式化；

包含文件系统的分区通常称为卷，卷可以是设备的一部分，或者是整个设备，或者是RAID集；

包含文件系统的每个卷也应该包含系统的文件信息，这些信息保存在设备目录或者卷目录表中，设备目录记录卷上的所有文件的信息；

存储结构

介绍了几种文件系统：

tmpfs – 用于快速临时I/O的基于内存的易失性FS
objfs –连接到内核内存以获取用于调试的内核符号
ctfs – 用于管理守护进程的合约文件系统
lofs – 环回文件系统允许访问一个 FS 来代替另一个
procfs – 进程结构的内核接口
ufs, zfs – 通用文件系统

目录概述

目录可以视为符号表，可以将文件名称转为目录条目；

对目录执行的操作：

  • 搜索文件
  • 创建文件
  • 删除文件
  • 遍历目录
  • 重命名文件
  • 遍历文件系统

单级目录

所有文件都包含在同一目录中；

缺点：名称必须是唯一的

两级目录

为每个用户创建一个用户文件目录（UFD），当用户工作时，搜索系统的主目录文件，主目录文件的每个条目指向一个用户的UFD；

用户名和文件名定义了路径名；

树形结构

允许用户创建子目录并相应地组织文件；有一个根目录，系统中的每个文件都有唯一的路径名；

目录只不过是一个文件，但是他是按照特殊方式处理的；所有目录都具有同样的内部格式；每个目录条目都有一位来将条目定义为文件（0）或者子目录（1）；通过特殊的系统调用，来创建和删除目录；

路径名：

  • 绝对路径名：从根目录开始；
  • 相对路径名：从当前目录开始；

无环图

树的结构不允许使用共享目录或共享文件；

使用无环图，即没有循环的图，可以允许共享子目录和文件；

两种共享方式：

  • 链接：此时是一个链接指向另一个文件；
  • 复制所有信息：此时两个文件是相等的；

删除方式：

  1. 只要有用户删除它，就删除它
    可能会留下悬空指针，指向不存在的文件，可以加一个反向指针，删除所有的指针；
  2. 保留文件，直到它的所有引用都被删除
    可以用一种机制来实现：
    • 创建所有引用的列表；
    • 只保留文件的引用计数；

通用图

为了保证正确性和性能，在图目录中要尽量避免出现环

我们如何保证没有循环？

  • 只允许链接到文件而不是子目录
  • 垃圾收集（Garbage collection）
    由于可能会出现自我引用的情况，需要垃圾收集，即：遍历整个文件系统，并标记所有可以访问的，接着，第二次遍历收集所有未标记的到空闲空间列表；垃圾收集很浪费时间，因此很少使用，但是他又是必须的；
  • 每次添加新链接时，使用循环检测算法来确定它是否正常

文件系统安装

每一个块设备上按一定格式建立起文件系统的时候，或者系统引导之初，设备上的文件和节点都还是不可访问的；也就是说，还不能按一定格式的路径名访问其中的某个特定的节点或文件；只有把它安装到计算机系统的某个文件系统中的某个结点上，才能使设备上的文件和节点成为可访问的；经过安装以后，设备上的文件系统就成为整个文件的一部分；

必须先挂载（Mount）文件系统才能访问它

一个未挂载的文件系统被挂载在一个挂载点上

  • 一般一个挂载点是一个空目录
  • 使用设备驱动读取文件目录并判定目录是否具备对应的文件格式

文件共享

多用户

在多用户系统上共享文件是可取的
共享过程需要进行文件保护

现在大多数系统都采用了文件所有者和组的概念：
所有者：可以更改属性和授予访问权限，拥有最高控制；
组：定义用户子集，一个组内的用户具有相同的访问权限；

远程文件系统

使用网络允许系统之间的文件系统访问

  • 通过文件传输协议FTP等程序手动
  • 自动、无缝地使用分布式文件系统
  • 半自动通过WWW万维网 + 浏览器
  • 云存储

客户机 - 服务器模型

客户机-服务器模式允许客户端从服务器挂载远程文件系统

服务器能服务多个客户端

NFS 是标准的UNIX客户端-服务器文件共享协议：
客户端和服务器的用户ID必须匹配

分布式信息系统

CIFS是标准的Windows协议

标准操作系统文件调用被转换成远程调用

故障模型

远程文件系统的故障很多，比如网络故障、服务器故障等，远程文件系统引入新的故障模式；

当一个客户机正在使用远程文件系统，他打开了远程主机的文件，突然，远程文件系统不再可用，所以客户机不应该将其按照本地文件系统丢失一样处理，而是应该在终止或者延迟操作；

大多数DFS协议（DFS为分布式文件系统）强制或者允许延迟操作，以等待服务器恢复；

从故障中恢复可能涉及关于每个远程请求状态的状态信息；

一致性语义

这些语义指定多个用户如何同时访问共享文件

由于磁盘I/O和网络延迟的原因，往往性能不好

Andrew File System (AFS) ：
实现了复杂的远程文件共享语义
AFS 具有会话语义，仅向文件关闭后启动的会话写入可见内容

Unix file system (UFS) :
写入正在打开的文件，该文件的其他用户可以立即看到该文件
共享文件指针，允许多个用户同时读写

保护

文件所有者/创建者应该能够控制：谁可以做什么

访问类型：

  • 读
  • 写
  • 执行
  • 附加
  • 删除
  • 访问文件属性列表

最常见的做法：根据用户身份控制访问，为每个文件和目录关联一个访问控制表（ACL），指定每个用户的名称以及允许访问的访问类型；

Unix / Linux上的三类用户

a) 所有者访问  7 -> RWX：1 1 1
b) 组访问  6-> RWX：1 1 0
c) 公开访问  1-> RWX：0 0 1

要求管理员创建一个组(唯一名称)，比如G，并向组中添加一些用户。

对于特定的文件(比如game)或子目录，定义适当的访问权限。
