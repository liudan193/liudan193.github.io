【10】大规模存储

大容量存储结构概述

磁盘

磁盘为现代计算机提供大量二级存储；

磁盘结构

盘片：两面都涂着磁性材料，通常为3.5英寸、2.5英寸和1.8英寸；

磁臂：将所有磁头作为一个整体一起移动；
磁头：附着在磁臂上；

磁道：圆形磁道；
扇区：每个磁道分为扇区；

柱面：同一磁臂位置的磁道的集合；

传输速率

每分钟转速（RPM）：每分钟转速；

传输速率：是驱动器和计算机之间数据流的速率；

定位时间（随机访问时间）：

  1. 寻道时间将磁臂移动到所需柱面的时间；
  2. 旋转延迟磁头旋转到所需扇区的时间；

性能

  • 传输速率 – 理论上 – 6 Gb/秒
    • 有效传输速率 – 实际上 – 1Gb/秒
  • 寻道时间从3毫秒~12毫秒 – 台式计算机9毫秒是常见的
    • 平均寻道时间：1/3轨道的寻道时间
  • 基于主轴速度的延迟： 1 / (RPM / 60) = 60 / RPM
    • 平均延迟 = ½ 延迟

访问延迟 = 平均访问时间 = 平均寻道时间+ 平均延迟

  • 对于最快的磁盘 3ms + 2ms = 5ms
  • 对于慢的磁盘 9ms + 5.56ms = 14.56ms

平均 I/O 时间 = 平均访问时间 + (传输量 / 传输速率) + 控制器开销

例如，在7200PRM磁盘上传输4KB块，平均寻道时间为5ms，传输速率为1GB/秒，控制器开销为0.1ms

平均 I/O 时间 = 5ms + 4.17ms + 0.1ms + 传输时间

传输时间= 4KB / （1Gb/s * 8Gb / GB * 1GB / 10242KB）
        = 32 / (10242) = 0.031 ms

4KB块的平均I/O时间= 9.27ms + .031ms = 9.301ms

其他

磁头碰撞：是由磁头与磁盘表面接触造成的；

磁盘可以是可移动的（软盘、CD、DVD、闪存驱动器、固态存储设备SSD）

驱动器通过I/O总线连接到计算机

  • 总线各不相同, 包括 EIDE, ATA, SATA, USB, 光纤通道, SCSI, SAS, Firewire
  • 计算机中的主机控制器使用总线与驱动器或存储阵列中内置的磁盘控制器通信（磁盘控制器通常具有内置缓存）

固态硬盘

  • 像硬盘一样使用非易失性存储器，许多技术变化

  • 比硬盘更可靠，它们没有移动部件

  • 但是要快得多

  • 它们没有寻道时间或延迟

  • 省电，容量较小，每MB更贵

  • 也许寿命更短

  • 总线可能太慢-> 例如直接连接到PCI

  • 直接替代磁盘驱动器

  • 新的缓存层

磁带

是早期的二级存储介质

  • 从开放式线轴发展到盒式磁带

相对永久并保存大量数据

访问时间慢

随机访问比磁盘慢约1000倍

主要用于备份、存储不常用介质、系统间传输介质

200GB到1.5TB 典型存储
常用技术是 LTO-5 和 SDLT

磁盘结构

磁盘驱动器

映射

磁盘驱动器可以被视为逻辑块的大型一维数组，其中逻辑块是最小的传输单位

  • 低级格式化在物理介质上创建逻辑块，可以通过低级格式化改变逻辑块的大小；

逻辑块的一维阵列依次映射到磁盘的扇区中；

扇区0是最外面柱面上第一个磁道的第一个扇区，映射按顺序通过该磁道，然后是该柱面中的其余磁道，然后从最外层到最内层通过其余柱面

理论上能进行逻辑地址到物理地址映射：（逻辑块号）<->（柱面号、磁道号、扇区号），但是有两个原因使得这个转换很难：

  1. 大多数磁盘都有一些缺陷扇区；
  2. 对于某些驱动器，每个磁道的扇区数并不是常量；

扇区数不是常量的详细解释

恒定线速度(CLV)

每个磁道的比特密度是均匀；
当磁头从外部磁道移动到内部磁道时，半径减小，能容纳的扇区越少；
驱动器会增加其旋转速度，以保持在磁头下移动的数据速率相同

例如：CD-ROM 和 DVD-ROM

恒定角速度 (CAV)

圆盘转速可以保持不变；
在这种情况下，位密度从内部磁道到外部磁道降低，保证每个磁道的扇区数相同，以保持数据速率恒定；

磁盘调度

介绍

如何衡量调度算法的效率：

  1. 访问时间；
  2. 带宽

调度算法是指：通过管理磁盘 I/O 请求的服务顺序来改善访问时间和带宽的算法

访问时间：磁臂移动磁头到包含目标扇区的柱面的时间；

磁盘带宽：是传输的总字节数除以第一次请求服务到最后一次传输完成之间的总时间

因为寻找时间 ≈ 寻找距离，可以用寻找距离去衡量寻道时间

请求队列

操作系统维护每个磁盘或设备的请求队列

空闲磁盘可以立即处理 I/O 请求，繁忙的磁盘意味着工作必须排队，当一个请求完成时，操作系统选择下一个待处理的请求

优化算法仅在队列存在时才有意义，没有等待的话，可以直接分配资源

磁盘I/O请求的来源有很多

  • 操作系统、系统进程、用户进程等

先来先服务（FCFS）

简单，但是效率低；

例如，现在访问柱面122，之后24，再之后123，那么，就会产生大摆动，很浪费时间；

最短寻道时间（SSTF）

最短寻道时间首先从当前磁头位置选择具有最短寻道时间的请求；

SSTF调度是SJF调度的一种变种；

可能会导致饥饿；

扫描算法（SCAN）

磁盘臂从磁盘的一端开始，向另一端移动，为请求提供服务，直到到达磁盘的另一端，在那里磁头反向移动并继续提供服务；

扫描算法有时被称为电梯算法；

但是请注意，如果请求的密度一致，则磁盘另一端的密度最大，等待时间最长；

循环扫描算法（C-SCAN）

磁头从磁盘的一端移动到另一端，在它移动时为请求提供服务，但是，和SACN不同，当它到达另一端时，它会立即返回到磁盘的开头，而不会在回程中处理任何请求；

可以提供比SCAN更均匀的等待时间；

可以认为是：将柱面视为从最后一个柱面到第一个柱面的循环列表；

C-LOOK

磁臂只到达每个方向的最后一个请求，然后立即反转方向，而不是首先到达磁盘的末端；

LOOK 是 SCAN 的 一 个 版 本 , C-LOOK 是C-SCAN的一个版本；

选择磁盘调度算法

SSTF 比较常见且性能最好；
SCAN和C-SCAN对于磁盘负载较重的系统性能更好，可以减少饥饿；
SSTF或LOOK都是默认算法的合理选择；

文件分配方法会影响对磁盘服务的请求：

  • 采用连续分配文件时，生成多个相近位置的磁盘请求，导致有限的磁头移动；
  • 采用链接或者索引的文件可能包括分散在磁盘上的多个块，导致更多的磁头移动；

磁盘调度算法应该作为操作系统的一个独立模块来编写，必要时可以用不同的算法来代替；

除了寻道距离之外，操作系统很难计算旋转延迟，因为现代磁盘没有透露逻辑块的物理地址；使用磁盘驱动器的控制器硬件内的磁盘调度算法，可以改善寻道时间和旋转延迟

SSD 通常使用 FCFS 策略
Linux Noop 调度程序使用 FCFS 策略，但修改它以合并相邻请求；
SSD 调度程序利用了这个特性，只合并相邻的写入请求，以 FCFS 顺序服务所有读取请求；

磁盘管理

磁盘格式化

低级格式化

一个新的硬盘是一个空白盘，只是一个磁性记录的盘子而已；

他必须先分成扇区，才能存储数据；

做个过程叫做低级格式化（或者物理格式化），低级格式化为每个扇区使用特殊的数据结构，填充磁盘；

  • 每个扇区由头部，数据区域，尾部组成；
  • 头部、尾部可以保存报头信息、数据和纠错码(ECC)；
  • 数据区域通常是512字节的数据，但可以选择；

逻辑格式化

要使用磁盘保存文件，操作系统仍然需要在磁盘上记录自己的数据结构，分为两步：

  1. 将磁盘分区（Partition）为一组或多组柱面，建立多个分区，每个分区都被视为一个逻辑磁盘；
  2. 逻辑格式化（Logical formatting）或创建制作文件系统，这一步可以将初始的文件系统数据结构）存储到磁盘上；

初始的文件系统数据结构包括空闲和已分配的空间，以及一个初始化为空的目录

为了提高效率，大多数文件系统将数据块分组集中到 簇（clusters） 中

  • 磁盘I/O以块为单位完成
  • 文件I/O以簇为单位中完成

原始 I/O（Raw I/O）

原始磁盘是指：将磁盘分区作为逻辑块的一个大型有序数组，而没有文件系统数据结构

原始I/O是指对原始磁盘的IO；

想要进行自己的块管理的应用程序的原始磁盘访问，使操作系统不受影响（例如数据库）；

原始 I/O 绕过所有文件系统服务，如缓冲区缓存、文件锁定、预取、空间分配、文件名和目录；

引导块

引导程序

为了运行计算机，在打开电源或者重启时，必须有一个初始程序来运行，这个初始化程序叫做引导程序；

引导程序找到磁盘中的操作系统内核，加载到内存并转到起始地址开始执行；

ROM中的小引导程序

引导程序一般存放在ROM中，因为ROM方便执行，但是不能修改；

所以一般都是在ROM中存放一个很小的引导程序，这个小程序的作用是从磁盘上调入完整的引导程序，由完整的引导程序用于引导启动系统；

这个完整的引导程序存储在磁盘固定位置上的 “启动块” 上中；

具有引导分区的磁盘称为引导盘(boot disk)或系统盘(system disk)

Windows示例

Windows将磁盘分为多个分区，有一个分区为引导分区，包含操作系统和设备驱动程序；

引导代码在磁盘的第一个扇区，称为主引导记录（MBR）；

MBR包含：

  • 一个表，用于列出所有的磁盘分区；
  • 一个标志，指示从哪个分区引导系统；

引导分区是包含操作系统的分区；
该分区的第一个扇区称为引导扇区；
而引导代码是在硬盘的第一个扇区，区分一下硬盘的第一个扇区和引导分区的第一个扇区；

硬盘维护

磁盘容易出现故障，一个或多个扇区出现缺陷；

用于处理坏块—备用扇区

一种策略是在格式化磁盘时扫描磁盘以查找坏块。发现的任何坏块都被标记为不可用，以便文件系统不会分配它们

控制器维护磁盘上的坏块列表

低级格式化还会留出操作系统不可见的备用扇区

交换空间管理

交换空间——虚拟内存使用磁盘空间作为主内存的扩展

  • 为虚拟内存系统提供最佳吞吐量
  • 交换空间在引导时重新初始化
  • 由于内存容量增加，现在不太常见

交换空间可以从普通文件系统中分割出来，或者更常见的是，它可以位于单独的磁盘分区(原始)，交换空间存储管理器对速度进行了优化

Linux例子

Linux的交换空间仅用于匿名内存（ Anonymous Memory ，如动态分配的堆、栈）

允许建立一个或多个交换区域，可以是常规文件系统上的交换文件，也可以是专用交换分区
