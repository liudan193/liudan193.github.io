【6】进程调度

|阅读量:

[TOC]

基本概念

CPU 调度是多道程序操作系统的基础。通过在进程间切换 CPU，操作系统可以使得计算机更加高效。

CPU-IO 执行周期

调度能成功的原因是因为进程具有的一个属性，即：进程执行包括周期进行 CPU 执行和 I/O 等待，进程在这两个状态之间不断交替。在最后一个 CPU 执行通过系统请求结束，以便终止执行。

CPU 调度程序

每当 CPU 空闲时，操作系统就应该从就绪队列中选择一个进程来执行。

抢占调度

CPU 调度的四种情况：

  • 当一个进程从运行状态切换到等待状态时，例如，I/O 请求，或者 wait() 调用；
  • 当一个进程从运行状态切换到就绪状态时，例如，当发生中断时；
  • 当一个进程从等待状态切换到就绪状态时，例如，I/O 完成时；
  • 当一个进程终止时；

定义：如果调度只发生在第 1 种和第 4 种情况下，则该调度方案是非抢占的；否则，调度方案称为抢占的（即，进程不是主动结束执行的，而是被其他进程抢占了执行的机会）。

在非抢占的调度下，一旦某个进程被分配到 CPU，该进程就会一直使用 CPU，直到它终止或切换到等待状态。

调度准则

也就是调度算法的评价指标

  • CPU 使用率：应使 CPU 尽可能忙碌，对于一个实际的系统，CPU 使用率应该是从 40%（轻负荷系统） - 90%（重负荷系统）；
  • 吞吐量：一个时间单位内进程完成的数量。如果 CPU 忙于进程，那么就称工作在完成。例如，对于长进程，吞吐量可能为每小时一各进程；
  • 周转时间：运行某个特定进程需要多长时间（即，从进程提交 到进程完成的时间段称为周转时间）。周转时间为所有时间段之和，包括等待进入内存、在就绪队列中等待、在 CPU 上执行和 I/O 执行；
  • 等待时间：在就绪队列中等待的时间；
  • 响应时间：从提交请求到产生第一次响应的时间；

几种调度算法

1. 先到先服务调度（FCFS）

很简单的一种方法，是非抢占式的，使用 FIFO 队列即可实现；

缺点：平均等待时间往往很长；

2. 最短作业优先调度（SJF）

SJF 介绍

CPU 空闲时，它会被赋给具有最短 CPU 执行的进程，同样时间的话，可以用 FCFS 来处理；

优点：SJF 的平均等待时间最小；

如何知道下次 CPU 执行的长度

SJF 的真正难点在于如何知道下次 CPU 执行的长度，

对于长期调度，可以将用户提交作业时指定的进程时限作为长度；

对于短期调度，可以试图近似 SJF 调度，即，预测下一次 CPU 执行的长度

一种预测方法为使用以前 CPU 执行的测量长度的指数平均，即：

$$\tau_{n+1} = \alpha t_{n} + (1-\alpha)\tau_{n}$$

抢占的和非抢占的两种 SJF

这种场景发生在当一个新进程到达就绪队列而以前的进程正在执行，但是这个新进程所需的 CPU 执行比正在执行的还要小的时候；

选择抢占式的 SJF 算法，会抢占该运行中的进程，这种方法有时候被称为最短剩余时间优先；

选择非抢占的 SJF 算法，则会允许当前运行进程先完成；

3. 优先级调度（PS）

优先级调度介绍

每个进程都有一个优先级与其关联，而具有最高优先级的进程会分配到 CPU，相同优先级的进程按照 FCFS 处理。

优先级调度也分抢占的和非抢占的。

主要问题 - 饥饿

优先级调度算法的一个主要问题是无穷阻塞或饥饿，就绪运行但是等待 CPU 的进程可以认为是阻塞的，优先级调度算法可以让某个低优先级进程无穷等待 CPU；

解决方案之一：逐渐增加系统中等待很长时间的进程的优先级，这种方法称为老化。

4.轮转调度（RR）

介绍

轮转调度算法是专门为分时系统设计的，类似于 FCFS 调度，增加了抢占以切换进程。具体而言，将一个较小的时间单元定义为时间量或者时间片。CPU 调度程序为每个进程分配不超过一个时间片的 CPU。

时间片大小一般为 10 ~ 100ms。

轮转调度算法是抢占的。

性能很大程度上取决于时间片的大小

RR 算法的性能很大程度上取决于时间片的大小。如果时间片很大，那么 RR 和 FCFS 一样了；如果时间片很小，那么 RR 算法会导致大量的上下文切换，因此，需要时间片远大于上下文切换时间。

5. 多级队列调度

介绍

进程分为不同组的情况下，可以与另一类调度算法。例如说，进程分为前台和后台进程，可以设置前台进程有更高的优先级。

多级队列将就绪队列分为多个单独队列，根据进程属性，如内存大小、进程优先级、进程类型等，永久分配一个进程到一个队列；每个队列有自己的调度算法。

6. 多级队列反馈调度

多级队列反馈调度算法允许进程在队列之间迁移。这种想法是：根据不同 CPU 执行的特点来区分进程。如果进程需要用到过多的 CPU 时间，那么他就会被移到更低的优先级队列。

这种方案将 I/O 密集型和交互进程放到更高优先级队列上，此外，在较低优先级队列中等待过长的进程会被移动到更高优先级队列，以防止饥饿的发生。

线程调度

竞争范围

进程竞争范围(Process-contention scope, PCS)：在实现多对一或多对多模型的系统中，线程库调度用户级线程到可用的LWP上。i.e. CPU竞争发生在“属于相同进程的线程”之间。

系统竞争范围(System-contention scope, SCS)：OS将内核线程调度到物理CPU上，该竞争发生在OS的所有线程中。采用一对一模型的OS的线程调度只采用SCS。

多处理器调度

多处理器调度的方法

非对称多处理：CPU 调度的一种方法是：让一个处理器（主服务器）处理所有调度决定、IO 处理等系统活动，其他处理器只执行用户代码；

对称多处理（SMP）：每个处理器自我调度。所有进程处于一个共同的就绪队列中或者每个处理器有他自己的私有调用队列。

处理器亲和性

如果进程迁移到另一个处理器上，那么会 1）原处理器缓存的内容无效，2）新处理器缓存应重新填充；

由于缓存的无效和重新填充的代价高，大多数 SMP 系统试图避免将进程从一个处理器移到另一个处理器，而是试图让一个进程运行在同一个处理器上。这称为处理器亲和性。

软亲和性：一个操作系统试图保持进程运行在同一个处理器上，但是不会保证这么做；
硬亲和性：提供系统调用以支持硬亲和性，从而允许某个进程运行在某个处理器子集上。

负载平衡

对于 SMP 系统，重要的是保持所有处理器的负载均衡，以便充分利用多处理器的优点。

负载平衡设法将负载平均分配到 SMP 系统的所有处理器。

对于私有就绪队列需要这么做，而对于公共队列的系统，负载平衡通常没有那么必要；

负载平衡的两种方法：

  • 推迁移：一个特定的任务，周期性地检查每个处理器的负载，如果发现不平衡，就通过将进程从超载处理器推到（push）空闲或者不太忙的处理器上；
  • 拉迁移：一个特定的任务，周期性地检查每个处理器的负载，如果发现不平衡，就通过空闲处理器从一个忙的处理器上拉（pull）一个等待任务。

多核处理器

当一个处理器访问内存时，他花费大量时间等待所需数据，这种情况被称为内存停顿。

为了弥补这种情况，许多硬件设计都采用了多线程的处理器核，即每个核会分配两个或者多个硬件线程，这样，一个线程停顿去等待内存的时候，该核可以切换到另一个线程。

处理器核的多线程可以分为两种方法：

  • 粗粒度：线程一直在处理器上执行，直到一个长延迟事件发生，由于长延迟事件造成的延迟，处理器应切换到另一个线程来开始执行；此时线程之间切换的成本较高，因为在另一个线程可以在处理器核上开始执行之前，应刷新指令流水线，新的线程开始执行之后，又会用指令来填充流水线；
  • 细粒度：在更细粒度级别上（通常在指令周期的边界上）切换线程，线程之间切换的成本更小；

实时 CPU 调度

软实时系统：不保证会调度关键实时系统，而只保证这类进程会优于非关键进程；

硬实时系统：有更严格的要求，一个任务应该再它的截止期限之前完成；

最小化延迟

事件延迟：从事件发生到事件得到服务的这段时间；

两种类型的延迟会影响实时系统的性能：

  • 中断延迟：从 CPU 收到中断到中断处理程序开始的时间。当一个中断发生时，应该先完成正在执行的指令，再确定中断的类型。然后，它应保存当前进程的状态，再采用特定的中断服务程序（ISR）来处理中断。
  • 调度延迟：调度程序从停止一个进程到启动另一个进程所需的时间量称为调度延迟。调研延迟分为两个阶段：冲突阶段（冲突阶段又分为两个部分：抢占在内核中运行的任何进程 & 释放高优先级进程所需的、低优先级占用的资源）和调度阶段。

优先权调度

用于实时操作系统的调度程序应支持抢占的、基于优先级的算法。

但是，只提供抢占的、基于优先级的算法仅仅能保证软实时功能，硬实时系统应进一步保证实时任务应在截止期限内得到服务，做出这样的保证需要附件的调度特征。该调度特征如下：

固定的处理时间 $t$、CPU 应处理的截止时限 $d$，和周期 $p$。三者满足：$0 \le t \le d \le p$
