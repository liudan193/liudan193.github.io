一、简要介绍

在 Python 中，并发并不是指在同一时刻有多个操作（thread、task）同时进行。在某个特定的时刻，它只允许有一个操作发生，只不过线程、任务之间会互相切换，直到结束。

并发有三种方式，包括多进程，多线程，多协成。

对于CPU密集型的程序，使用多进程；对于 IO 密集型的，使用多线程。

二、GIL（Global Interpreter Lock）

GIL 的功能是：在 CPython 解释器中执行的每一个 Python 线程，都会先锁住自己，以阻止别的线程执行。

在CPython中，每一个Python线程执行前都需要去获得GIL锁 ，获得该锁的线程才可以执行，没有获得的只能等待 ，当具有GIL锁的线程运行完成后，其他等待的线程就会去争夺GIL锁，这就造成了，在Python中使用多线程，但同一时刻下依旧只有一个线程在运行 ，所以Python多线程其实并不是「并行」的，而是「并发」，也叫做伪并行，可以通过下面这张图来解释。

简而言之，GIL 下的 Python 多线程其实是多个线程在顺序执行而不是并行的执行。
