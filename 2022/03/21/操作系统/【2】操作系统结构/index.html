【2】操作系统结构

操作系统的服务

1. 操作系统为程序和用户提供执行程序和服务的环境

操作系统有一组服务，为用户提供功能：

  1. 用户界面 —— 几乎所有的操作系统都有用户界面（UI）；
  2. 程序执行 —— 系统能加载程序到内存并加以运行，并且可以结束执行，包括正常和不正常（并给出错误）；
  3. I/O操作 —— 程序运行可能需要I/O，这些I/O可能涉及文件或者设备；
  4. 文件系统操作 —— 用户对文件系统特别感兴趣。程序需要读写文件和目录，创建和删除、搜索、列出文件信息、管理权限；
  5. 通信 —— 进程可以在同一台计算机上或通过网络在不同计算机之间交换信息；
  6. 错误检测 —— 操作系统需要不断检测和更正错误；

特别地解释一下操作界面

1. 用户与操作系统界面 —— CLI（△命令解释程序）

  • 有时在内核中实现，有时由系统程序实现
  • 对于具有多个可选命令解释程序的系统，解释程序称为外壳 (shell)
  • 命令解释程序的作用：获取并执行用户指定的下一条命令
  • 有时是内置命令，有时只是程序名称，后者不必理解什么命令，只需要用命令来识别文件以装入内存并执行；如果是后者，添加新功能不需要修改shell

2. 图形用户界面 —— GUI

允许操作系统使用基于鼠标的窗口和菜单系统作为接口，使图标代表文件、程序目录等

3. 触摸屏

2. 存在另一组操作系统功能，通过资源共享来确保系统本身的高效运行

  1. 资源分配 —— 当多个用户或多个作业同时运行时，必须为每一个用户分配资源；
  2. 记账 —— 记录用户使用资源的类型和数量；
  3. 保护和安全 —— 对于存储在多用户或联网计算机系统中的信息，用户可能希望控制该信息的使用。并发进程不应相互干扰；

系统调用

1. 系统调用的介绍

系统调用提供操作系统的接口，通常用高级语言（C/C++）编写，每个系统调用都有可能遇到错误情况

通常由程序通过高级应用程序编程接口（API）访问，而不是直接使用系统调用

常见的 API：

  1. 用于 Windows 的 Win32 API
  2. 适用于 POSIX 系统的 POSIX API（包括几乎所有版本的 UNIX、Linux 和 Mac OS X）
  3. 用于 Java 虚拟机（JVM）的 Java API

2. 系统调用的实现

系统调用都有一个相关索引号，系统调用接口会根据索引号来建立一个索引列表；

然后，系统调用接口调用操作系统内核中的所需系统调用，并返回系统调用状态与任何返回值。

调用者无需知道如何实现系统调用（只需遵循API）

系统调用根据使用计算机的不同而不同，除了所需的系统调用，还要提供其他信息，即传递参数，信息的具体类型和数量根据特定操作系统和调用而有所不同；

传递参数的三种方式：

  1. 寄存器 —— 最简单，但是有时候会出现参数数量会多于寄存器数量的问题
  2. 内存 —— 参数存在内存的块或表中，而块或表的地址通过寄存器来传递，Linux和Solaris就采用这种方法
  3. 堆栈 —— 参数也可通过程序放在或压入到堆栈, 并通过操作系统弹出

内存和堆栈方法不限制传递参数的数量或长度

3. 系统调用的类型

进程控制

  • 创建进程，终止进程
  • 结束、 中止
  • 加载、执行
  • 获取进程属性、设置进程属性
  • 等待时间
  • 等待事件、信号事件
  • 分配和释放内存
  • 如果出错则转储内存
  • 用于确定错误的调试器，单步执行
  • 用于管理进程间共享数据访问的锁

文件管理

  • 创建文件、删除文件
  • 打开、关闭文件
  • 读、写、重新定位
  • 获取和设置文件属性

设备管理

  • 请求设备、释放设备
  • 读、写、重新定位
  • 获取设备属性、设置设备属性
  • 逻辑附加或者分类设备

信息维护

  • 获取时间或日期、设置时间或日期
  • 获取系统数据，设置系统数据
  • 获取和设置进程、文件或设备属性

通信

  • 创建、删除通信连接
  • 如果消息传递模型获取到主机名或进程名，则发送、接收消息（从客户端到服务器）
  • 共享内存模型创建并访问内存区域
  • 传送状态信息
  • 附加或分离远程设备

保护

  • 控制对资源的访问
  • 获取和设置权限
  • 允许和拒绝用户访问某些资源

系统程序

系统程序为应用程序的开发和执行提供了一个便利的环境

可以分为：文件管理、状态信息、通信、程序语言支持、程序加载和执行、后台服务、应用程序等几类

文件管理
创建、删除、复制、重命名、打印、转储、列出、操作文件和目录

状态信息
从系统那里得到日期、时间、内存或磁盘空间的可用数量、 用户数或其他状态信息。
还有一些则更为复杂，可提供详细的性能、登录和调试信息
通常， 这些信息经格式化后，再打印到终端、输出设备或文件
一些系统支持注册表——用于存储和获取配置信息

文件修改
用于创建和修改文件的文本编辑器
用于查找文件内容或执行文本转换的特殊命令

程序语言支持
编译程序、 汇编程序、 调试程序和解释程序

程序加载与执行
绝对加载程序、重定位加载程序、链接编辑器
覆盖式加载程序、高级语言和机器语言的调试程序

通信
提供在进程、用户和计算机系统之间创建虚拟连接的机制
允许用户向彼此的屏幕发送消息、浏览网页、发送电子邮件、远程登录、将文件从一台机器传输到另一台机器

后台服务
在启动系统时启动
一些执行完任务后就终止
一些会一直运行到系统停机
提供磁盘检查、进程调度、错误记录、打印等功能
在用户上下文中运行，而不是内核上下文
称为服务、子系统或守护进程

应用程序
不属于系统
由用户运行
通常不被认为是操作系统的一部分
通过命令行启动（鼠标单击等）

操作系统的设计与实现

操作系统的设计与实现没有完整的解决方案，不同的操作系统内部结构可能会有很大差异。

通过定义目标和需求开始设计

  • 受所选硬件与系统类型的影响
  • 用户目标和系统目标
    • 用户目标——操作系统应该便于使用、易学、可靠、安全和快速
    • 系统目标——操作系统应该易于设计、实现和维护，以及灵活、可靠、正确和高效

策略与机制的分离，机制决定怎么做，策略决定做什么。

  • 策略与机制的分离是一个至关重要的原则，如果以后要更改策略的话可以提供最大的灵活性（例如 ：计时器）

分析和设计操作系统是软件工程中极具创造性的任务

实现： 通常是混合语言，主体是C语言

操作系统的结构

1. 简易结构 MS-DOS

以最少的空间提供最多的功能，不分模块。

没有分功能的接口和层次，比如没有内核态和用户态的区分

任意性使得MS-DOS易受错误（或恶意）程序的伤害，从而导致用户程序出错导致整个系统的崩溃

2. 非简易结构 UNIX

UNIX 由内核和系统程序两个独立的部分组成。

  • 系统程序
  • 内核
    • 物理硬件之上和系统调用接口之下的所有部分组成内核
    • 内核通过系统调用提供文件系统、CPU调度、内存管理等操作系统功能
  • 非简易结构，但是不完全分层

3. 层次结构（Layered Approach）

  • 分为很多层，每一层都建立在较低层之上
  • 底层（layer 0）为硬件，最高层（N 层）为用户界面
  • 通过模块化，所选的层次要求每层只能调用更低层的功能和服务
  • 易于管理，主要优点在于构造和调试的简单化，但不允许跨层访问，效率较低，功能层的决定（即某个程序应该放在哪层）比较困难

4. 微内核系统结构（Microkernels）

  • 尽可能多地将功能从内核移入用户空间，将非基本部分从内核中移走，并将它们实现为系统程序或用户程序，这样得到了更小的内核；
  • 哪些放到内核，哪些放到用户空间并没有定论，但是，微内核通常包括最小的进程和内存管理，以及通信功能；
  • 优点：
    • 更容易扩展操作系统，所有新的服务可以加在用户空间
    • 更容易将操作系统移植到新的平台
    • 更安全可靠（内核模式下运行的代码更少）
  • 缺点：
    • 增加了用户空间与内核空间通信的性能开销

5. 模块化架构（Modules）

  • 使用面向对象的方法生成模块化的内核
  • 每个核心组件都是独立的
  • 每个模块都可以通过已知接口与其他模块通信
  • 每个模块都可以根据需要在内核中加载
  • 与分层系统类似，允许内核提供核心功能，也能动态地实现特定的功能，但更加灵活，Linux、Solaris使用这种结构
  • 类似于微内核，核心模块只有核心功能和其他模块加载和通信的相关信心，但是更加高效，因为不需要消息传递来通信

6. 混合系统

现代操作系统大都不是一种单一的、严格定义的结构

混合系统结合多种方法来满足性能、安全性和可用性需求
Linux和Solaris是复合系统，采用模块化结构，新的功能可动态添加到内核
Windows 大多是复合的，加上用于不同个性的子系统微内核

  1. Mac OS X结构
    采用混合结构，是一个分层的系统
  2. ios
    移动操作系统，以Mac OS X为基础，增加了功能
  3. Android
    基于经过修改的Linux内核

操作系统的调试

调试就是查找和更正系统错误，以及性能优化

  • 操作系统生成包含错误信息的文件日志
  • 应用程序的故障会生成核心转储文件，可以使用gdb指令进行程序调试
  • 操作系统故障会生成包含内核内存状态的崩溃转储文件
  • 除了崩溃外，调试还可以改善系统性能

性能优化：

  • 通过消除瓶颈提高性能
  • 操作系统必须提供计算和显示系统行为度量的方法
    • top 命令(w, uptime, ps, etc.)
    • Windows 任务管理器
    • 日志文件(log file)

操作系统的生成

操作系统通常被设计成能运行在任何一类计算机上，必须为每个特定的计算机配置对应的系统（不同的CPU, RAM, 设备等）

系统生成（SYSGEN）程序获取有关硬件系统的特定配置信息

  • 用于构建系统特定的编译内核或系统优化

当硬件发生变化时，操作系统重新生成配置，无需重新编译操作系统源码

系统引导

在系统上初始化电源时，从固定的内存位置开始执行

  • 固件ROM（ Firmware ROM ）用于保存初始引导程序

操作系统对应硬件可用，这样硬件才能启动它

  • 一小段代码 —— 存储在 ROM 或者 EEPROM 中的引导加载程序定位内核，将其加载到内存中，然后执行
  • 有时候是两步过程，其中固定位置的引导块由ROM代码加载，从磁盘加载引导加载程序

通用引导加载程序GRUB允许从多个磁盘、版本和内核选项汇总选择内核

内核加载，然后系统运行
