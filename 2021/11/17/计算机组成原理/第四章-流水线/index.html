流水线

发表于2021-11-17|更新于2021-11-17|计算机组成原理
|阅读量:

1. 流水线概述

  • 流水线是一种实现多条指令重叠执行的技术。

  • 优点：

    1. 改善了吞吐率（并行执行）
    2. 未改变单个指令的执行时间

  • 速度提升倍率 ≈ 流水线中步骤的个数

2. 计算机指令的流水

首先，MIPS指令的处理包括五个步骤：1).从指令存储器中读取指令；2).指令译码的同时读取寄存器；3).执行操作或计算地址；4).从数据存储器中读取操作数；5).将结果写回寄存器。

2.1 计算机指令流水线的性能

流水线的性能受限于最慢的一个步骤，因为所有的流水级都只花费一个时钟周期时间，而时钟周期需要设置为每个时钟周期的时间一致，所以只能按照最慢的那一个来。

比如说取值、译码（并读取）、执行、读数、写回五个步骤中，最慢的读数为200ps，其余均为100ps，那么时钟周期需要设置为200ps才能满足每个是周期结束的时候，流水线的各个步骤都已经完成，可以进行下一个步骤。

在理想的情况下（即每个步骤时间相同），流水线上的指令执行时间为：

指令执行时间（流水） = 指令执行时间（非流水） / 流水线级数

2.2 面向流水线的指令集设计

第一：所有的MIPS指令的长度都一样。这样简化了取指和译码阶段；

第二：指令格式少，并且每一条指令中的源寄存器字段位置都是相同的。这样就可以在第二级译码的同时，开始读取寄存器堆；

第三：存储器操作数仅出现在存取指令中。这样的话就可以利用别的指令在进行执行(Ex)的步骤时，就计算存储器地址，如果不这样的话，假如 add 指令也要执行读取存储器，那么则需要将执行步骤拆分为计算地址和执行两个步骤；

第四：所有操作数在存储器中对齐。这样的话，则不需要担心一个指令需要访问两次存储器了。

3. 流水线冒险

3.1 结构冒险

因为缺乏硬件而导致指令不能在预定的时钟周期内执行的情况。

比如，第一条指令的访问存储器阶段和第四条指令的取指阶段同时访问了一个存储器，那么流水线就会发生结构冒险。

3.2 数据冒险

数据冒险发生在由于一条指令执行，需要用到上一条或者前面几条指令结束之后得出的数据，但是前面的指令还未完成的时候。

解决方案：

  1. 硬件方面进行阻塞（stall），插入空泡（bubble）或者软件层面程序员自己插入空指令 nop；

  2. 重新安排指令的顺序，比如书中 P188 例题；

  3. 前推方法：从内部寄存器中提前提取出数据，一旦ALU产生了数据之后，就将ALU的结果传输给下一条指令需要用到的地方。

    前推方法中，如上例的add -> sub命令不需要插入空泡，只要加入旁路路径即可，而像lw -> sub这种取数-使用型数据冒险则不得不要加入一个空泡了；

3.3 控制冒险

当决策依赖于前面beq或者bne的分支结果的时候遇到控制冒险，此时还不知道要不要执行该指令。

解决方案：

  1. 阻塞，插入空泡，直到结果出来为止；

  2. 假定分支不发生；

  3. 缩短分支的延迟：提前分支决策，即将计算分支目标地址和判断分支条件提前；

  4. 动态分支预测：观察上一次执行该指令时分支是否发生，如果上一次发生，就从上一次方式的地方开始取新的指令；

    实现动态分支预测的一种方法就是采用分支预测缓存，分支预测缓存是一小块按照分支指令的地址低位索引的的存储器区，用于记录上次分支是否发生；

4. 指令级并行

4.1 指令级并行

加深流水线的级数，例如将五级流水线分为六级流水线。

4.2 多发射

一个周期内将多个指令发射到多套部件，这样的缺点是需要额外的工作让机器同时运转并将负载传到下一个流水级；

实现多发射流水线有两种方式，静态多发射和动态多发射，其区别是将主要工作分给编译器还是硬件来做。
