简化模型搭建过程 – ModuleList 和 Sequential

ModuleList 和 Sequential 是用来简化模型的搭建过程的，会自动的将需要训练的参数加入到网络中。

两者都可简化模型的构造过程，但是Sequential不用写forward函数（内部实现了），ModuleList要自己定义类，并且自己实现forward函数。

ModuleList：

他是一个储存不同 module，并自动将每个 module 的 parameters 添加到网络之中的容器，这个容器可以通过索引来获得某个模型。

你可以把任意 nn.Module 的子类 (比如 nn.Conv2d, nn.Linear 之类的) 加到这个 list 里面，他拥有的方法和 Python 自带的 list 一样，无非是 extend，append 等操作。但不同于一般的 list，加入到 nn.ModuleList 里面的 module 是会自动注册到整个网络上的，同时 module 的 parameters 也会自动添加到整个网络中。若使用python的list，则会出问题，所以要用nn.ModuleList 来写。

使用ModuleList需要自定义一个类，在 __init__ 函数中写入MouduleList，在 forward 里面使用ModuleList里面的网络（或层）。

对于网络中不同层设置不同的学习率

为不同子网络设置不同的学习率，这在finetune中经常用到。

示例代码中，1e-5是默认的学习率，如果未对某个参数指定学习率则使用默认学习率，如示例中net网络中的features部分使用1e-5的学习率；也可以自己指定学习率，如classifier则使用1e-2 。

调整学习率

调整学习率两种方法：

  1. 修改optimizer.param_group中的参数
  2. 直接新建一个优化器（更推荐的方法，由于optimizer开销很小，不会浪费较多的资源）

nn.funtional

  • nn.xxx 和 nn.functional.xxx 的实际功能是相同的，当需要底层操作的时候可以采取funtional自己写

  • nn.xxx 和 nn.functional.xxx 的运行效率也是近乎相同
