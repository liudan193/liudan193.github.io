浮点数

发表于2021-10-17|更新于2021-10-19|计算机组成原理
|阅读量:

1. 规则化（Normalize）

就像是十进制里面，科学技术法前面的数字要求小于10，大于0一样（例如5.631 * 10^15为规范化的，而155.25 * 10^15为非规范化的），二进制也有一样的规定，小数点前面的数字大于0，小于2（二进制里面也就是1），即为规则化的。

2. 浮点数在计算机中的存储（遵循IEEE 754标准）

2.1 在寄存器内的存储方式

单精度浮点数（float）：

s E F
符号 指数 尾数
1位 8位 23位

双精度浮点数（double）：

s E F F
1位 11位 20位 32位

则该浮点数代表的数值为：(-1)^s * F * 2^E （错的，暂时这么写）。

2.2 优化1 -> 扩大尾数F

为了扩大数值表示范围，IEEE 754 标准使用了两个优化方法。

第一个即为扩大尾数F，由于规范化之后，二进制的非小数部分均为1，因此中这个1可以省略，如1.00可以只存一下.00，1.11可以只存一下.11。

因此，让寄存器中将剩下的23位（双精度为52位）只做小数位，即小数点后面的那些，默认加上前面的1.，这样可以扩大为24位（双精度为53位）。

经过优化1之后的数值为：(-1)^s * (1+F) * 2^E（也是错的，后面还有一层优化）。

2.3 优化2 -> 扩大指数E

由于指数只有8位，只能表示 2^(0000 0000) ~ 2^(1111 1111)的范围，而指数也有负数，因此采取了减去一个偏置的方法（单精度浮点数里面的偏置为127），即如果在寄存器中存储一个 0000 0001 (十进制的1)，则代表的是 1-127 = -126，则这个数指数部分大小为 2^(-126)。双精度的bias为1023。

经过优化2之后的数值为：(-1)^s * (1+F) * 2^(E-bias)（这个是最终对的版本）。

最后，32位浮点数表达的数字范围为 ±1.1111 1111 * 2^(+127)。

可能会奇怪为什么255-127=128，但是范围中最大指数部分为2的127次，那是因为指数为0或者255代表特殊的值。

2.4 特殊的数字存储形式

正如上面写的，指数位为0或者255的时候，代表特殊的数值（仅针对单精度(32位)的是255，双精度为2047）。

指数 位数 数值
0 0 0
0 非0 非规范化数
255 0 无穷
255 非0 NaN（Not a Number）

2.5 优化后使用浮点数举例

将-0.75（十进制）放在32位的寄存器中：

  1. 将十进制转换成规范化的二进制，-0.75（十进制） -> -3/4（十进制），-3/4（十进制） -> -11（二进制）/2（十进制）^2 -> -0.11（二进制） -> -1.1（二进制）*2^(-1)；
  2. 按照 (-1)^s * (1+F) * 2^(E-127) 放在32位的寄存器中
    符号位：负的为1；
    尾数：1.1只取小数点后面的 1；
    指数：126 - 127 = -1，因此为126；
符号位 尾数 指数
1 0000 0000 0000 0000 0000 001 0111 111 0
