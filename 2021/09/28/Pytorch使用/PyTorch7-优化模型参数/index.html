优化模型参数

发表于2021-09-28|更新于2021-09-28|PyTorch
|阅读量:

前置代码

获得训练集、测试集和建好的模型。

设置超参数

超参数是自己手动设置的可调参数，它让你控制模型的优化过程。不同的超参数值会影响模型的训练和收敛速度。

现在我们设置下面几种超参数：

  • Number of Epochs — 总共迭代的次数
  • Batch Size — 每次取的数量详细解释
  • Learning Rate — 学习率，太大容易找不到最优点，太小容易降低学习速度，消耗时间更多

添加优化的循环

一旦我们确定我们的超参数，我们就可以训练我们的模型了。优化循环的每次迭代被称为epoch。

每个epoch包括两个主要部分：

  • The Train Loop — 训练集上的循环，为了收敛到最优的参数（权重和偏置等）
  • The Validation/Test Loop — 测试集上的循环，输出每次的结果，为了确保模型效果确实是在提升

增加一个损失函数

常见的损失函数包括，nn.MSELoss(均方差)，MSE主要解决回归问题，还有nn.NLLLoss(负对数似然)，主要解决分类问题。

nn.CrossEntropyLoss结合了 nn.LogSoftmax and nn.NLLLoss。

为我们的模型选择nn.CrossEntropyLoss作为损失函数。

优化过程

优化是在每个训练步骤中调整模型参数以减少模型误差的过程。

优化算法定义了这个过程是如何执行的（在这个例子中我们使用随机梯度下降）。所有优化逻辑都封装在optimizer(优化器)对象中。

在这里，我们使用 SGD 优化器；此外，PyTorch 中有许多不同的优化器可用，例如 ADAM 和 RMSProp ，它们更适用于不同类型的模型和数据。

我们通过传入需要被训练的参数和学习率来初始化优化器。

全部实现

我们定义了循环优化的函数 train_loop 和根据测试数据评估模型性能的函数 test_loop。

我们初始化损失函数和优化器（optimizer），并将其传递给 train_loop 和 test_loop。 随便增加 epochs 的数量以跟踪模型的改进性能。

你可能已经注意到，最初模型不是很好(没关系！)。 尝试运行更多epoch的循环或调整learning_rate更大一些。它也可能是我们选择的情况下，该模型配置对这类问题不是最优的那个。

补充

Batch Size的解释

参考：谈谈深度学习中的 Batch_Size

首先，为什么需要有 Batch_Size 这个参数？

Batch 的选择，首先决定的是下降的方向。如果数据集比较小，完全可以采用全数据集 （ Full Batch Learning ）的形式。

这样做至少有 2 个好处。其一，由全数据集确定的方向能够更好地代表样本总体，从而更准确地朝向极值所在的方向。其二，由于不同权重的梯度值差别巨大，因此选取一个全局的学习率很困难。 Full Batch Learning 可以使用 Rprop 只基于梯度符号并且针对性单独更新各权值。

对于更大的数据集，以上 2 个好处又变成了 2 个坏处。

其一，随着数据集的海量增长和内存限制，一次性载入所有的数据进来变得越来越不可行。其二，以 Rprop 的方式迭代，会由于各个 Batch 之间的采样差异性，各次梯度修正值相互抵消，无法修正。这才有了后来 RMSProp 的妥协方案。

既然 Full Batch Learning 并不适用大数据集，那么走向另一个极端怎么样？

所谓另一个极端，就是每次只训练一个样本，即 Batch_Size = 1。这就是在线学习（Online Learning）。

使用在线学习，每次修正方向以各自样本的梯度方向修正，横冲直撞各自为政，难以达到收敛。

可不可以选择一个适中的 Batch_Size 值呢？

当然可以，这就是批梯度下降法（Mini-batches Learning）。因为如果数据集足够充分，那么用一半（甚至少得多）的数据训练算出来的梯度与用全部数据训练出来的梯度是几乎一样的。
